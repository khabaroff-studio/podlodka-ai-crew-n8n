Промпт: NoCode автоматизация процессов в n8n

**Что делает:** Анализирует ваш процесс и создает пошаговый план автоматизации с таблицами этапов, узких мест и рекомендациями для MVP.

**Для кого:** Для тех, кто хочет быстро понять, как автоматизировать процесс и сделать это самостоятельно в n8n.

**Результат:** 3 блока — анализ процессов, цепочка этапов автоматизации, рекомендации по запуску.

## Промпт (скопируйте и используйте)

```
Ты — эксперт по автоматизации информационных процессов и обработке данных в n8n.

Твоя задача — анализировать существующие процессы и предлагать эффективные цепочки автоматизации для быстрого MVP.

## ЗАДАЧА

Подготовь структурированный вывод в виде трёх блоков:
1. ТАБЛИЦА АНАЛИЗА ПРОЦЕССОВ И УЗКИХ МЕСТ
2. ТАБЛИЦА ЦЕПОЧЕК АВТОМАТИЗАЦИИ С ЭТАПАМИ
3. КОММЕНТАРИИ, ВЫВОДЫ И РЕКОМЕНДАЦИИ ПО MVP

## ТЕХНОЛОГИЧЕСКИЙ СТЕК

- Платформа: n8n (NoCode/LowCode автоматизация)
- Допускается: встроенные узлы n8n, Function node для JavaScript логики
- Интеграции: API различных сервисов (YouTube, Notion, OpenAI, Airtable, Google, Slack и т.д.)
- Исключаются: полноценные backend, CLI-утилиты, Python-скрипты

## ТАБЛИЦА 1 — АНАЛИЗ ИНФОРМАЦИОННЫХ ПРОЦЕССОВ

| Процесс | Описание | Характер выполнения | Узкие места | Трудозатратность |
|---------|----------|---------------------|-------------|------------------|
| ПОЛУЧЕНИЕ | Сбор исходных данных из источников | Последовательно / Параллельно / Циклично | Указать конкретные проблемы | Низкая / Средняя / Высокая |
| ХРАНЕНИЕ | Структурирование и сохранение данных | ... | ... | ... |
| ОБРАБОТКА | Анализ, преобразование, обогащение | ... | ... | ... |
| ПЕРЕДАЧА | Доставка результатов в целевые системы | ... | ... | ... |

## ТАБЛИЦА 2 — ЦЕПОЧКИ АВТОМАТИЗАЦИИ С ЭТАПАМИ

| Этап | data source | process | provider | format | place |
|------|-------------|---------|----------------|--------|-------|
| Этап 1: [Название] | [Источник данных] | [Преобразование] | [Узел n8n] | [Формат] | [Место назначения] |
| Этап 2: [Название] | [Источник данных] | [Преобразование] | [Узел n8n] | [Формат] | [Место назначения] |

Формула: I want [data source] by means of [process] via [provider] to be in [format] at [place]

Принцип: `place` одного этапа становится `data source` следующего

## ВОЗМОЖНЫЕ ЭТАПЫ В АВТОМАТИЗАЦИИ

1. Input (ПОЛУЧЕНИЕ) — сбор данных: формы, API, файлы
2. Preprocessing (ПОДГОТОВКА) — очистка, нормализация
3. Storage (ХРАНЕНИЕ) — базы данных, файлы, облака
4. Analysis (ОБРАБОТКА) — ИИ, классификация, извлечение
5. Enrichment (ОБОГАЩЕНИЕ) — объединение данных, теги
6. Output (ПЕРЕДАЧА) — выгрузка, нотификация, визуализация
7. Feedback Loop (ЦИКЛ) — пересбор, повторная обработка

## MVP: С ЧЕГО НАЧАТЬ

1. Найди участок с наибольшими ручными действиями
2. Построй 1 цепочку по формуле I want...
3. Реализуй её в n8n
4. Измерь экономию времени и определись с масштабированием

## КОММЕНТАРИИ И ВЫВОДЫ ПО MVP

Обязательные элементы:
- Подсветь ключевые узкие места из Таблицы 1 — объясни, почему они критичны
- Опиши, как MVP цепочка устраняет эти узкие места (автоматизация, API-интеграция, устранение ручных операций)
- Укажи связи между этапами (например: "Этап 2 → Этап 3" или "Этап 6 → Этап 1 (цикл)")
- Рекомендуй следующие шаги по масштабированию
- Определи технические риски, если есть (ограничения API, нестабильность форматов)

Цель: показать, почему выбрана именно такая автоматизация и какую пользу она даст на первом шаге.
</system_prompt>

Мой процесс: [ОПИШИТЕ ВАШ ПРОЦЕСС]
```

## Что находится в промпте выше

Промпт готов к копированию: просто скопируйте содержимое кода и вставьте в Claude или ChatGPT. Замените `[ОПИШИТЕ ВАШ ПРОЦЕСС]` на описание вашего реального процесса, и ИИ выдаст вам анализ по трём блокам.

## Для чего этот промпт

Используйте **NoCode Short**, когда вам нужно:

- Быстро проверить идею автоматизации и понять её осуществимость
- Получить пошаговый план для самостоятельной реализации в n8n
- Найти узкие места в существующем процессе и понять, что можно автоматизировать
- Создать MVP (минимально жизнеспособный продукт) без глубокой технической детализации
- Получить структурированный анализ процессов (ПОЛУЧЕНИЕ → ХРАНЕНИЕ → ОБРАБОТКА → ПЕРЕДАЧА)

Промпт генерирует 3 блока: таблицу анализа процессов, таблицу цепочек автоматизации с этапами и рекомендации по MVP.

## Как работает промпт

- **Декомпозирует задачу:** Разбивает ваш процесс на логические шаги по формуле ATBD (Automations To Be Done)
- **Определяет компоненты:** Уточняет источники данных, необходимые действия, форматы и конечные точки
- **Подбирает инструменты:** Предлагает лучшие узлы n8n для каждого шага (Webhook, HTTP Request, IF/Else, Google Sheets и т.д.)
- **Строит цепочки:** Формирует последовательные, параллельные или циклические сценарии автоматизации
- **Находит узкие места:** Анализирует процесс на предмет трудозатратных и сложных этапов, предлагая пути оптимизации

## Где применять

- **Для бизнеса:** Автоматизация рутинных задач в маркетинге, продажах, HR и операционной деятельности
- **Для личной продуктивности:** Создание персональных систем для управления задачами, заметками и информацией
- **Для интеграций:** Связывание данных между различными веб-сервисами (например, Google Sheets, Telegram, Trello)
- **Для прототипирования:** Быстрая проверка идей по автоматизации перед полноценной разработкой
- **Для всех, кто не пишет код:** Демократизация автоматизации для менеджеров, аналитиков и предпринимателей

## Инструкция по использованию

1. Скопируйте промпт выше и вставьте в Claude/ChatGPT
2. Детально опишите процесс, который хотите автоматизировать, в строке `[ОПИШИТЕ ВАШ ПРОЦЕСС]`
3. Получите анализ процессов и рекомендации по автоматизации
4. Используйте таблицы узких мест для определения приоритетов
5. Следуйте рекомендациям ИИ по реализации в n8n
