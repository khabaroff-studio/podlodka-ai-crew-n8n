Промпт: aрхитектор No-Code Проектов"

**Что делает:** Создает полное техническое задание (ТЗ) для workflow в n8n с описанием узлов, обработки ошибок, безопасности и рисков.

**Для кого:** Для заказчиков, которые передают задачу фрилансеру или команде разработки и нужен детальный документ для исполнителя.

**Результат:** 6-раздельная техническая документация — от общей информации до рисков и неясностей. Полный промпт для n8n с примерами и таблицами.

## Для чего этот промпт

Используйте **NoCode Long**, когда вам нужно:

- Передать задачу по автоматизации фрилансеру или в команду разработки
- Создать полное техническое задание с детальной архитектурой
- Спроектировать сложную интеграцию с множеством сервисов и условий
- Документировать обработку ошибок, retry-логику и fallback-сценарии
- Получить готовый документ для долгосрочного проекта с поддержкой

Промпт генерирует 6-раздельную техническую документацию с описанием nodes, безопасности, отказоустойчивости и рисков.

**Альтернатива:** Если вам нужен быстрый план для самостоятельной реализации MVP без глубокой технической детализации — используйте ```Prompt NoCode Short```.

## Как работает промпт

- **Анализирует вашу цель:** ИИ задает уточняющие вопросы, чтобы понять суть задачи.
- **Декомпозирует процесс:** Разбивает большую задачу на конкретные сценарии, триггеры и действия.
- **Проектирует архитектуру:** Создает логику workflow для n8n, определяя узлы и условия.
- **Предусматривает риски:** Продумывает обработку ошибок, повторные попытки (retry) и альтернативные пути выполнения.
- **Формирует ТЗ:** Упаковывает все в структурированный технический документ с описанием узлов, данных и API.

## Где применять

- **Создание ТЗ для разработчика:** Передайте задачу по автоматизации фрилансеру или в команду разработки, минимизируя неточности.
- **Проектирование сложных интеграций:** Свяжите несколько сервисов (например, CRM, Slack, Airtable, Telegram, PostgreSQL) в единый workflow в n8n.
- **Документирование и рефакторинг:** Опишите и улучшите существующие автоматизации, чтобы сделать их надежнее и понятнее.
- **Самостоятельная разработка:** Используйте сгенерированное ТЗ как четкий план действий для самостоятельной настройки workflow в n8n.

## Твоя миссия:
- Создавать полное описание рабочих процессов, интеграций и поведенческих логик в форме технического документа

- Формировать документацию так, чтобы она могла быть непосредственно реализована в n8n

## Алгоритм работы (цепочка рассуждений):
1. **Понять**: определи цель системы и ожидаемое поведение пользователей и автоматизаций
2. **Выделить**: раздели проект на workflow'ы, триггеры, действия, условия, пользователей
3. **Оркестрировать**: построй логику в виде n8n workflow'ов, с учетом параллелизма и условий
4. **Оптимизировать**: указывай способы обработки ошибок, повторов, ветвлений, таймаутов
5. **Кодифицировать**: где нужно, добавь вставки JavaScript-кода (только в Code nodes)
6. **Оценить**: проверь, каким образом вся логика может быть поделена на переиспользуемые блоки. Определи, какие workflow'ы можно выделить как sub-workflows, а какие являются одноразовыми
7. **Оформить**: выведи структурированный документ с описанием workflow'ов и параметров

### 5. Безопасность и отказоустойчивость

### 6. Риски и неясности

## Применение и советы

- Для наилучшего результата будьте максимально конкретны в своем запросе. 
- Промпт оптимизирован для n8n, поэтому описывайте задачу с учетом особенностей платформы (nodes, workflows, Code nodes).
- Этот промпт идеально подходит для создания документации *до* начала разработки, что помогает избежать ошибок и переделок.
- Для работы с этим промптом требуется мощная модель ИИ (например, GPT-4 или Claude 3.5 Sonnet), так как она должна понимать сложную системную логику и удерживать большой объем контекста.
